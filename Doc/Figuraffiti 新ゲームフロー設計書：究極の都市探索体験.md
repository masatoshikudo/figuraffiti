# Figuraffiti 新ゲームフロー設計書：究極の都市探索体験

**作成者:** Manus AI  
**作成日:** 2026年3月7日

本ドキュメントは、改善提案（曖昧なマップ、FTF制度、デジタルタギング、組み合わせアンロック）を統合した、Figuraffitiの新しいプレイヤー体験（UX）の時系列フローを定義したものです。

---

## 1. プレイヤーの体験フェーズ（全体像）

プレイヤーの体験は、以下の4つのフェーズで構成されます。単なる「移動と記録」から、**「情報収集 → 推理と探索 → 達成とマーキング → 次なる謎」**というゲームループへと進化します。

1. **フェーズ1：噂の探知（Discovery）** - SNSやマップで「存在の匂い」を感じ取る
2. **フェーズ2：現地での探索（Hunt）** - 曖昧なエリアとヒントを頼りに、自らの目で探す
3. **フェーズ3：発見と刻印（Tagging）** - QRを読み取り、自分の痕跡（タグ）を残す
4. **フェーズ4：共鳴と次なる謎（Unlock）** - コレクションが揃うことで、隠された存在が浮かび上がる

---

## 2. 各フェーズの具体的なゲームフロー

### フェーズ1：噂の探知（Discovery）
**目的:** 「どこかに何かがある」という期待感を煽り、ユーザーを街へ向かわせる。

*   **1-1. SNSでの新着通知:**
    公式アカウント（Instagram/TikTok）が「新たな存在が街に現れた」ことを告知します。この時、正確な場所は明かさず、キャラクターの視点から見た風景（例：室外機の裏から見上げた空）だけを投稿します。
*   **1-2. マップの確認（Web/アプリ）:**
    ユーザーがアプリを開くと、マップ上に**「半径50m〜100mの曖昧なサークル（出現エリア）」**が表示されています。正確なピンはありません。
*   **1-3. ヒントの解読:**
    サークルをタップすると、キャラクターのシルエットと「断片的なヒント（謎めいたテキストや、周囲のテクスチャ写真）」が表示されます。ユーザーは「渋谷のあの路地裏ではないか？」と推理を始めます。
*   **1-4. FTF（First to Find）の確認:**
    まだ誰も見つけていない場合、画面には大きく**「#1 未発見（FTFチャンス！）」**と表示され、一番乗りの競争心を強烈に煽ります。

### フェーズ2：現地での探索（Hunt）
**目的:** 「行けば見つかる」作業を排除し、「見つけた！」というアハ体験（カタルシス）を生み出す。

*   **2-1. エリアへの到着:**
    ユーザーはマップのサークルエリアに到着します。しかし、ナビゲーションはそこまでです。
*   **2-2. 都市の余白の観察:**
    ユーザーはスマホから目を離し、物理的な空間を注意深く観察し始めます。配電盤の裏、ガードレールの隙間、看板の影など、普段は見過ごす「都市の余白」を探します。
*   **2-3. フィギュアの発見:**
    苦労の末、小さな立体グラフィティ（Figuraffiti）を発見します。この「自力で見つけ出した」瞬間の喜びが、ゲームの最大の報酬となります。

### フェーズ3：発見と刻印（Tagging）
**目的:** 発見の証をデジタル空間に刻み込み、自己顕示欲と所有欲を満たす。

*   **3-1. QR/NFCの読み取り:**
    フィギュアに付いているQRコードをスマホで読み取ります。
*   **3-2. 劇的なリザルト画面:**
    *   **あなたが最初の発見者の場合:** 画面が派手に演出され、「FTF（First to Find）達成！」のバッジが付与されます。
    *   **2番目以降の場合:** 「あなたは #42 番目の発見者です」とシリアルナンバーが表示されます。
*   **3-3. デジタル・タギング（発見報告）:**
    ユーザーは、そのキャラクターのページに自分の「デジタルタグ（10文字程度のストリートネームや一言）」と、発見時の写真/動画URLを残します。
*   **3-4. キャラクターページの変容:**
    キャラクターページは、訪れたユーザーたちのデジタルタグで埋め尽くされた「デジタル上の壁（グラフィティ・ウォール）」として機能し、コミュニティの熱量が可視化されます。

### フェーズ4：共鳴と次なる謎（Unlock）
**目的:** 1つの発見で終わらせず、次なる探索への強力な動機付けを行う。

*   **4-1. コレクションの確認:**
    マイページで、自分が集めたキャラクター（シリアルナンバーとFTFバッジ付き）を眺めます。
*   **4-2. 隠し要素のアンロック（共鳴）:**
    例えば、「キャラクターA」と「キャラクターB」の両方を発見した瞬間、画面にノイズが走り、**「2つの存在が共鳴し、新たな影が浮かび上がった…」**という演出とともに、マップ上にこれまで存在しなかった**「シークレットサークル」**が新たに出現します。
*   **4-3. 次のループへ:**
    ユーザーは「このシークレットを見つけたい」「SNSで自慢したい（考察を語りたい）」という欲求に駆られ、再びフェーズ1へと戻ります。

## 3. UI/画面設計の変更イメージ（フェーズ1対応版）

既存の要件定義の画面構成をベースに、ゲーム性を高めるためのUI変更点です。

| 画面名 | 従来の設計（要件定義） | 新ゲームフローでの設計（変更点） |
| :--- | :--- | :--- |
| **マップ画面** | 承認済みスポットをピンで表示。 | ピンを廃止し、**「出現エリア（サークル）」**で表示。未発見（FTF待ち）のサークルは色を変えて目立たせる。 |
| **スポット詳細** | 住所、説明、関連メディアを表示。 | 住所を隠し、**「キャラクター視点の写真」や「暗号テキスト」をヒントとして表示**。 |
| **QR読み取り先（キャラページ）** | 情報、ストーリー、発見ログを一覧表示。 | **「あなたが #◯番目の発見者です」**という劇的な演出。訪問者たちの**「デジタルタグ（短いテキスト）」が壁の落落書きのように重なって表示**されるUI。 |
| **マイページ** | 自分の発見報告一覧。 | 取得した**FTFバッジ**の数、発見したキャラクターの**シリアルナンバー（#◯）**を強調したコレクションラック風のUI。 |

## 結論

この新しいゲームフローにより、Figuraffitiは単なる「フィギュアの場所共有マップ」から、**「現実の都市を舞台にした謎解きと、デジタル上の陣取りゲーム（タギング・FTF競争）」**へと変貌します。

ユーザーは「運営が用意したコンテンツを消費する」のではなく、「自らの足と目で都市を解読し、自らの痕跡をデジタル空間に刻み込む」という、ストリートカルチャーの本質に根ざした強烈なゲーム体験を得ることができます。
