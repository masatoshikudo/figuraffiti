# AhhHum SNSバズ戦略マスタードキュメント

本ドキュメントは、`AhhHum` の認知獲得と行動転換を目的とした SNS 戦略の正本である。  
`AhhHum ニッチマーケ戦略マスタードキュメント.md` を上位方針とし、その中でも特に **TikTok / Instagram / X / Threads / YouTube Shorts** の運用に絞って実務設計へ落とし込む。

本ドキュメントは、`AhhHum figuraffiti 関係定義ドキュメント.md` を前提とする。  
以後、`AhhHum` は物販ブランド、Webマップ / ゲームサイトは参加導線、`figuraffiti` は内部戦略・設計用語として扱う。  
なお、`figuraffiti` という語は外部公開表現には用いず、AhhHum の体験特徴へ翻訳して扱う。

---

## 1. 戦略の結論

AhhHum の SNS 戦略は、次の一文に要約される。

> **都市の違和感を「事件」として見せ、共鳴した人を AhhHum の静かな世界観と、その参加導線である Webマップ / ゲームサイトへ送客する。**

重要なのは、SNS 上で売り込むことではない。  
SNS はあくまで **認知と感情喚起の入口** であり、AhhHum は物販ブランド、Webマップ / ゲームサイトはその参加導線である。

したがって、SNS でやるべきことは以下の 3 つに絞る。

1. **事件化する**  
   日常の風景に生まれた小さな違和感を、誰でも気になる形に翻訳する

2. **プロセスを見せる**  
   完成品ではなく、都市に意味が立ち上がるまでの過程を追わせる

3. **静かな導線へ送る**  
   バズの熱量を AhhHum の世界観と、参加導線である Webマップ / ゲームサイトへ接続する

このとき最重要なのは、**SNSで伸びること** と **AhhHum が強くなること** を混同しないことである。  
再生だけが伸びても、都内23区の重点区で探索行動が起きなければ、ニッチマーケ戦略上は成功ではない。

---

## 1.5 この戦略の補助線

この文書は単独では成立しない。  
上位の `AhhHum ニッチマーケ戦略マスタードキュメント.md` と整合させるため、以下を前提とする。

1. **SNSは市場エントリーの入口であり、収益化の入口ではない**
2. **初期市場は都内23区、ただし運用は重点区集中**
3. **AhhHum は物販ブランドであり、SNS はその価値を支える参加導線へ送る翻訳装置である**
4. **物販告知は参加体験の熱量が立ってから接続する**
5. **関東展開は23区モデルの再現性確認後に行う**

## 2. 前提原則

### 2.1 SNS の役割

SNS は販売チャネルではなく、以下の役割を持つ。

- 違和感の発見を可視化する
- 重点区で探索が実際に動いていることを可視化する
- 現地に行く可能性のある人を炙り出す
- Webマップ / ゲームサイトへの導線を作る

### 2.2 ブランドとの関係

`AhhHum` は静かで匿名的なブランドであり、SNS の派手さと直接同一化させない。  
そのため、SNS 上では **演者アカウント** と **ブランドアカウント** の役割を分ける。

| アカウント | 役割 | トーン | 主目的 |
| :--- | :--- | :--- | :--- |
| 覆面アーティスト / 演者 | 事件化、制作プロセス、熱量の可視化 | 人間味、偏愛、狂気、実況性 | 新規認知 |
| AhhHum 公式 | 世界観の保存、参加方法の整理、商品価値の蓄積 | 静寂、匿名、文脈、余白 | 行動転換、保存、再参加 |

### 2.3 初期アカウント運用ルール

スキルに従い、立ち上げ初期は AI にアカウントの方向性を学習させる期間とみなす。

- 発信ジャンルをぶらさない
- 動画フォーマットとサムネイルの型を固定する
- 初期に身内シェア、相互フォロー、無関係コラボをしない
- 1アカウント1役割を徹底する
- 非ターゲットに最適化しすぎず、必ず AhhHum の参加体験へ帰着させる
- 内部用語 `figuraffiti` を公開投稿で前面に出さない

### 2.4 バズ戦略の失敗パターン

AhhHum では、以下は「伸びても失敗」とみなす。

- 再生は多いが、都内23区の本命層に保存されていない
- コメントは多いが、現地に行きたい人が増えていない
- 話題になったが、`違法っぽい何か` としか認識されていない
- 演者のキャラクターだけが伸び、AhhHum 側に世界観資産が残っていない
- 投稿本数は多いが、重点区の運用密度より先に露出だけが広がっている

## 3. ターゲット設計

### 3.1 本命ターゲット

最初に狙うのは、以下のような人たちである。

- 都内23区で徒歩移動が多い 20-30代
- 街、建築、アート、裏路地、都市文化系のコンテンツを日常的に見る
- 「買うこと」より「知っている場所が増えること」に価値を感じる
- SNS 上では静かだが、刺さるものは保存し、後で見返す
- 休日に 30-90 分程度の回遊行動が取れる

### 3.2 SNS で広く取る周辺層

SNS では、いきなりこの本命層だけを狙うとスケールしにくい。  
そのため、入口は以下の周辺層まで広げてよい。

- 都市の不思議・違和感動画が好きな人
- ミニチュア、フィギュア、ストリートアート、隠し要素が好きな人
- 「こんなところに何であるの？」に反応する人
- 都会の散歩・探検・デート先探しをしている人

ただし、最終的な導線は必ず `都市探索` と `発見記録` に収束させる。

## 4. プラットフォーム別戦略

### 4.0 各プラットフォームの出口

媒体ごとの役割は違うが、出口が曖昧だとニッチ戦略とズレる。  
そこで、各媒体で最終的に取るべき行動を明確にする。

| プラットフォーム | 1次ゴール | 2次ゴール | 最終出口 |
| :--- | :--- | :--- | :--- |
| TikTok | 視聴維持 | プロフィール遷移 | AhhHum 認知の定着と参加導線への送客 |
| Instagram Reels | 保存 | プロフィール回遊 | Stories / プロフィールから Webマップ導線へ |
| Instagram Feed | 世界観理解 | 保存 | 再参加・指名検索 |
| X | 考察参加 | プロフィールクリック | 世界観への理解と議論蓄積 |
| Threads | 軽い関心喚起 | リンク遷移 | Webマップ / ゲームサイト流入 |
| YouTube Shorts | 資産再利用 | チャンネル認知 | 長期接触の土台 |

### 4.0.1 Phase別のSNS目的

ニッチマーケ戦略との整合性を保つため、SNS の目的もプロダクトフェーズに従って変える。

| フェーズ | SNSの目的 | やること | やらないこと |
| :--- | :--- | :--- | :--- |
| Phase1 | 認知と行動転換の検証 | 違和感の事件化、重点区の探索導線、発見ログの可視化 | 強い販売訴求 |
| Phase2 | 体験拡張としての収益接続 | 物理キーや記念商品の意味づけ、消失 / 再発見の魅力訴求 | 商品単体の量販訴求 |
| Phase3 | 共創の質を上げる | 設置文脈、審査、アンバサダー価値の可視化 | 誰でも自由に置ける印象づけ |
| Phase4 | IPの翻訳と拡張 | コラボ、都市回遊企画、ブランド横断の物語化 | 本体の文脈を捨てた大衆迎合 |

### 4.1 TikTok

主目的は **新規認知の最大化**。

- 最重要指標: 再生維持率、視聴完了率、プロフィール遷移率
- 形式: 15-45秒中心、必要に応じて60秒超でプロセス版も試す
- 主役: 覆面アーティスト / 演者
- 見せるもの:
  - こんな場所にあったのか、という違和感
  - 設置までの葛藤
  - 見つかった瞬間
  - 消失、再発見、都市の反応
- CTA:
  - 基本は `プロフィールへ`
  - 一部の勝ち投稿だけ `重点区の探索導線` へ接続する

### 4.2 Instagram Reels / Feed

主目的は **世界観の定着と保存行動の獲得**。

- 最重要指標: 保存率、プロフィール回遊率、DM反応
- Reels: TikTok の勝ち動画を再編集して展開
- Feed: 静止画、場所の文脈、断片的な考察、余白あるコピー
- Stories: 発見報告、再掲、投票、次の導線づくり
- CTA:
  - Reels は `保存`
  - Feed は `世界観理解`
  - Stories は `マップ回遊`

### 4.3 X

主目的は **考察・議論・リアルタイム性の獲得**。

- 最重要指標: プロフィールクリック率、保存、引用、言及
- 投稿構成: 逆三角形
- 強いテーマ:
  - 都市の余白とは何か
  - なぜその場所に置いたのか
  - 撤去、消失、発見に関する現場のリアリティ
  - ストリートカルチャーと違法性の境界
- CTA:
  - 即流入よりも `理解の深化`
  - 固定ポストで AhhHum の世界観と参加導線へ送る

### 4.4 Threads

主目的は **X の勝ち筋再利用と外部誘導の補助**。

- 過去に反応の良かった短文・長文の再編集を流す
- リンク付きでも誘導しやすい特性を活かし、Webマップ / ゲームサイト導線を補強する

### 4.5 YouTube Shorts

主目的は **資産化と横展開**。

- TikTok の勝ち動画を再利用
- 検索文脈や長期発見に備える
- 将来的にロング動画やドキュメンタリーへの入口にする

## 5. コンテンツ設計

### 5.1 コンテンツの基本思想

SNS で伸びるのは「情報」ではなく「感情」である。  
したがって AhhHum の投稿は、単なる作品紹介ではなく、**日常の事件化** と **意味が立ち上がるまでの過程の可視化** を軸に設計する。

### 5.2 主要コンテンツピラー

1. **違和感発見系**
   こんな場所に何かある、という驚きを見せる

2. **制作プロセス系**
   造形、準備、設置前の思考、失敗、葛藤を見せる

3. **探索反応系**
   誰かが見つけた、迷った、考察した、更新した瞬間を見せる

4. **都市文脈系**
   なぜこの場所なのか、その地形や景色とどう関係するのかを語る

5. **消失と再覚醒系**
   なくなった、でも記録が残る、再び見つかった、という物語を見せる

6. **思想と言語化系**
   AhhHum の哲学、都市の余白論、探索の意味を X や Threads で言語化する

### 5.3 投稿比率の初期目安

- 40%: 違和感発見系
- 25%: 制作プロセス系
- 15%: 探索反応系
- 10%: 都市文脈系
- 10%: 思想と言語化系

### 5.4 重点区コンテンツ戦略

ニッチマーケ戦略では、初期市場は都内23区だが、実運用は重点区集中である。  
したがって SNS でも、`東京全体` を雑に語るのではなく、重点区ごとの密度感を作る。

- 1本の動画で広域を見せすぎない
- 重点区ごとに `この区には見えない物語がある` という印象を作る
- 同じ区内で複数の投稿接点を作り、発見可能性を蓄積させる
- 区ごとに景色、トーン、歩き方、文脈を変える
- 23区全域を均等に扱うのは、重点区モデルが成立してからにする

### 5.5 供給制約を前提にした投稿設計

この戦略の弱点は、運用密度より先に SNS 露出だけが先行することにある。  
よって、投稿本数は多ければ良いのではなく、**現地供給と運用体制に見合う範囲で増やす**。

- 重点区内の実スポット数より、話題だけを先行させない
- 発見後にマップで何も起きない状態を作らない
- 消失・撤去・更新の運用が追いつかない時は投稿頻度を落とす
- 露出より `再び参加したくなる密度` を優先する

## 6. ショート動画の勝ちパターン

### 6.1 基本構成

スキルのテンプレートに従い、ショート動画は以下の流れで作る。

1. **冒頭2秒のフック**
   事件性のある一言で止める

2. **展開**
   何が起きているのか、なぜ気になるのかを見せる

3. **伏線回収**
   実は AhhHum の参加体験や文脈設計の話だったと回収する

4. **行動喚起**
   コメント、保存、フォロー、マップ遷移を促す

### 6.2 AhhHum で使えるフック例

- 「こんな場所に、なぜこれがあるのか。」
- 「渋谷の道端に、意味不明なものを置きました。」
- 「誰にも気づかれないはずだったのに、見つかりました。」
- 「昨日まであったのに、消えました。」
- 「見つけた人だけ、次の意味が分かります。」
- 「この街、見えていない場所が多すぎる。」
- 「ただ見るだけで終わらないように、してあります。」

※ 強い言葉は使ってよいが、スキルに従い **事実に紐づく表現のみ** を使う。  
釣りや誇張だけで視聴を取ることは避ける。

### 6.2.1 フックの使用基準

強いフックは有効だが、AhhHum では以下の条件を満たす時だけ使う。

- 後半で必ず意味を回収できる
- 違法性や迷惑行為の推奨と誤読されない
- 演者の人格ではなく、都市体験に関心が戻る
- 投稿後にブランド側で説明不能な誤解を生まない

逆に、以下の表現は避ける。

- 単なる炎上誘発
- 実態以上に危険そうに見せる表現
- 誰でも真似できる違法行為の指南に見える表現

### 6.3 動画フォーマット例

#### フォーマットA: 違和感発見
- 0-2秒: 強い一言
- 2-8秒: 何が起きているかの全景
- 8-18秒: 寄りの映像、場所の文脈
- 18-25秒: 伏線回収
- 25-30秒: 「見つけられる人いる？」で終える

#### フォーマットB: 制作プロセス
- 0-2秒: 問題提起
- 2-10秒: 制作中の葛藤
- 10-25秒: 置くまでの流れ
- 25-35秒: 完成後の静かな映像
- 35-45秒: 次回予告

#### フォーマットC: 発見報告
- 0-2秒: 「ついに見つかりました」
- 2-10秒: 場所はぼかして反応を見せる
- 10-20秒: NFC / 発見の瞬間
- 20-30秒: 「次は誰が見つける？」で締める

## 7. X 長文戦略

### 7.1 基本構成

X の長文投稿は、スキルに従い **逆三角形** で書く。

1. 誰でも入れる入口
2. 共感できる具体例
3. AhhHum 独自の視点
4. 結論

### 7.2 AhhHum に向くテーマ

- 「街は情報で埋まりすぎていて、余白が足りない」
- 「なぜ説明しない体験の方が記憶に残るのか」
- 「違法性を煽らずにストリートの美学を残せるか」
- 「発見した人がログを残すと、場所はどう変わるか」
- 「都市を歩く意味を、どうやって再発明するか」

### 7.3 固定ポストの役割

固定ポストには、以下を満たす投稿を置く。

- AhhHum が何者か分かる
- どう参加すればよいか分かる
- 世界観が伝わる
- Webマップや導線がある
- 初見でも離脱しない

## 8. 投稿運用ルール

### 8.1 初期 30 投稿ルール

最初の 30 投稿は、アルゴリズム学習期間とみなし、以下を徹底する。

- フォーマットを固定する
- ジャンルをぶらさない
- 売り込み投稿を混ぜない
- 無関係な日常投稿をしない
- 毎回、同じターゲット感情を狙う

### 8.2 投稿頻度の目安

- TikTok / Reels: 週4-7本
- X: 週5-10投稿
- Instagram Feed: 週2-3投稿
- Stories: ほぼ毎日
- Threads: 週3-5投稿
- YouTube Shorts: 週2-4本

ただし、この頻度は **重点区での供給と運用が回る場合の上限目安** である。  
素材の薄い状態で本数だけ増やすと、ニッチ戦略で重視する文脈密度が崩れる。

### 8.3 コメント運用

- コメント欄は「考察」と「発見欲」を増幅させる場として使う
- 答えを全部明かさない
- 見込み客の意図信号を拾う
- 荒れそうな論点はブランドアカウントで過剰応答しない
- 内部用語をそのまま公開コメントで定着させようとしない

### 8.4 ロケーション情報の扱い

SNS は拡散力が強いため、場所の出し方を誤るとニッチ戦略の `曖昧サークル` と衝突する。  
したがって、位置情報の扱いは以下を原則とする。

- 重点区までは示してよいが、即座に特定できる情報は出しすぎない
- 危険な導線、迷惑行為を誘う情報は隠す
- 発見の快感を損なうほど答えを明かさない
- 特定が早すぎて密度戦略が崩れる場合は投稿タイミングを遅らせる

## 9. KPI 設計

### 9.1 認知KPI

- 再生数
- 平均視聴維持率
- 視聴完了率
- 保存率
- プロフィール遷移率
- 指名検索数

### 9.2 行動転換KPI

- プロフィールリンククリック率
- Webマップ流入数
- マップ内サークル詳細タップ率
- 現地訪問意向コメント数
- NFC タッチ率
- 重点区別の流入比率

### 9.3 見込み客シグナル

スキルに従い、単なる再生よりも **意図信号** を重視する。

- 「どこですか」
- 「行ってみたい」
- 「今度探す」
- 「これ東京のどの辺ですか」
- 「どうやって記録するんですか」
- 「設置したい」

ただし、意図信号も質で分ける必要がある。

- `弱い意図信号`: 「気になる」「すごい」「行ってみたい」
- `強い意図信号`: 「今週行く」「場所予想したい」「設置したい」「キーが欲しい」

### 9.4 判断基準

伸びた投稿を評価するときは、以下の順で判定する。

1. マスに届いたか
2. 本命ターゲットが反応したか
3. 行動転換が起きたか
4. ブランドの核を壊していないか

### 9.5 SNS成功の定義

AhhHum における SNS 成功は、次の条件を満たすことである。

1. 投稿が伸びる
2. 本命ターゲットに保存される
3. 都内23区の重点区で探索行動が起きる
4. 発見ログと再参加が増える
5. その結果として、後から収益商品が意味を持つ

この順序を崩し、`再生 -> 商品告知` に短絡すると、ニッチマーケ戦略との整合性が崩れる

## 11. 実行ロードマップ

### フェーズ1: アルゴリズム学習

- アカウント設計
- 投稿フォーマット固定
- 初期 30 投稿の投入
- 反応パターンの把握
- 重点区ごとの反応差の把握

### フェーズ2: 勝ち筋の特定

- 最も保存率・遷移率が高いテーマを抽出
- フックの勝ちパターンを特定
- 重点区の投稿テーマを深掘る
- 演者側とブランド側の役割分担を確定する

### フェーズ3: 行動転換の強化

- Webマップ導線の明確化
- 発見報告の二次拡散設計
- 発見ログを SNS コンテンツへ再循環
- 重点区から隣接区へ広げても密度感が保てるか検証する

### フェーズ4: 収益接続

- 物理キーや限定商品の告知を解禁
- ただし売る理由は「モノ」ではなく「体験拡張」に限定する

### フェーズ5: 関東展開の翻訳

- 東京で勝った投稿パターンをそのまま複製しない
- 神奈川、千葉、埼玉では土地文脈ごとに翻訳する
- `東京でバズったから関東でもいける` という短絡を避ける
