# AhhHum 管理運用MVP確定仕様書

## 目的
- 本ドキュメントは、AhhHum Phase1 MVP における `管理者によるスポット運用` の確定仕様をまとめたものとする。
- 対象は `スポット作成 / 公開 / NFC token 運用 / 発見後の扱い / 再設置の前提` までとする。

## 1. スポット作成運用
- スポットを作成できるのは `管理者のみ` とする。
- 作成時の必須項目は `スポット名 / 文脈メモ / 設置位置 / NFC token` とする。
- MVPでは `下書き保存` は持たず、管理者作成時にそのまま登録する。
- 同じ場所への再設置でも、MVPでは `毎回新規spot` として扱う。

## 2. 公開タイミング
- 開発環境では、スポット作成後 `即公開` とする。
- 本番環境では、`NFC token発行から1時間後` に公開する。
- 公開制御には `visible_after` を使う。
- `新スポット公開` ティッカーは、`visible_after` 到達時を公開時刻として扱う。
- 管理画面では、公開前スポットを `#公開待ち` として判別できるようにする。

## 3. NFC token 運用
- token発行は `スポット作成と同時` に行う。
- NFC書き込みは `管理画面外の一般アプリ` で行う。
- タグ上書き時は `パスワード必須` の運用を想定する。
- tokenは `1 spot につき有効なものを1つ` とする。
- 発見記録成功後、そのtokenは `使用済み` となり再利用できない。
- 書き込みミス時は、将来 `旧token無効化 + 新token再発行` を行える余地を残すが、MVPでは詳細運用の確定までは行わない。

## 4. 発見後 / 持ち去り後の扱い
- 発見記録が成功したspotは `履歴入り` とする。
- 履歴入りしたspotは `マップ上から非表示` にする。
- 持ち去られたか不明なまま残るspotは、`半年` を目安に公開対象外とする。
- 関係ない人に持ち去られたケースは、MVPでは `放置` とする。

## 5. 管理画面で見えるべき状態
- `#公開待ち`
- `#公開中`
- `#履歴入り`
- `#期限切れ`

## 6. 再設置の前提
- 同じ場所への再設置は、MVPでは `新規spot + 新規token` とする。
- ただし将来拡張では、同一地点の系譜を束ねる余地を残す。
- 同一地点の判定バッファは `半径10m` を前提とする。
