# AhhHum 最新ゲームフロー設計書：共創型・都市探索IPの完全版

本ドキュメントは、既存の「究極の都市探索体験」案と、その後に整理された持続運営・共創・マネタイズ設計を統合した、AhhHum の最新ゲームフロー定義である。  
`AhhHum figuraffiti 関係定義ドキュメント.md` を前提とし、AhhHum は物販ブランド、`figuraffiti` はその中核表現・参加メカニズム、Webマップ / ゲームサイトはその参加導線として扱う。  
目指すのは、単なる場所共有サービスではなく、**都市探索のスリル、発見の証明、共創による拡張、IPとしての持続性**が一体化した参加体験である。

画面ルート、画面ID、モジュール、Phaseごとのスコープ分割は、`AhhHum Webアプリ全体設計マスタードキュメント.md` を優先する。

## 1. コアコンセプト

Webマップ / ゲームサイトは、プレイヤーを「地図を見る人」ではなく、**都市を読み解き、自分の痕跡を残し、やがて都市そのものを共創する人**へ変えていく参加導線である。

この体験は、以下の4つを同時に成立させることで完成します。

1. **探索性:** 正確な答えを最初から与えず、自力で見つける喜びを最大化する。
2. **証明性:** 発見順、FTF、デジタルタグによって「見つけた事実」を残せる。
3. **循環性:** 発見で終わらず、次の謎や次の探索動機が生まれる。
4. **持続性:** 消失や過疎を仕組みで吸収しつつ、共創と物販へ段階的に接続する。

## 2. 3種のフィギュアとエコシステム

AhhHum のエコシステムは、役割の異なる3種類のフィギュアで構成される。これにより、初期は運営主導で品質を保ちつつ、中長期ではユーザー共創型へ拡張できる。

| 種別 | 役割と機能 | 取得方法 | マネタイズ |
| :--- | :--- | :--- | :--- |
| **① 運営設置用** | 都市に最初の「種」を蒔く非売品。世界観と品質基準を定義する。 | 運営のみ | - |
| **② 物理キー用** | 自宅観賞用フィギュア。休眠スポットの正確な位置を解放する「再覚醒キー」として機能。 | ECサイトで購入 | 直接販売 |
| **③ 設置用** | ユーザーが運用対象エリアに設置できる権利付きフィギュア。NFCに設置者の文脈や想いを持たせられる。 | ECサイトで購入 | 直接販売 |

## 3. プレイヤー体験フェーズ

プレイヤー体験は、以下の5フェーズで設計する。  
単なる「移動と記録」ではなく、**「噂を知る → 現地で探す → 発見を刻む → 次の謎が生まれる → 世界が循環する」**というゲームループに変換する。

### フェーズ1：噂の探知（Discovery）
**目的:** 「どこかに何かがある」という期待感と、重点区で探索が実際に動いていることを伝える。

* **1-1. グローバル・ティッカー:** マップ画面の隅に「User_A just found #42 in Shibuya」のような発見ログが流れ、世界が動いていることを伝える。
* **1-2. 曖昧なサークル表示:** マップ上には正確なピンではなく、半径50m〜100m程度の出現エリアのみを表示する。
* **1-3. ヒントの提示:** サークルをタップすると、キャラクターのシルエット、断片的なテキスト、周辺テクスチャ写真、キャラクター視点の風景などが表示される。
* **1-4. 鮮度の確認:** 「Last Seen: 2 hours ago」のような最終発見時刻を表示し、「まだある可能性が高い」と判断できるようにする。
* **1-5. FTF状態の可視化:** 未発見スポットには「#1 未発見」「FTFチャンス」と表示し、一番乗りの競争心を刺激する。

### フェーズ2：現地での探索（Hunt）
**目的:** スマホを見る時間ではなく、街を観察する時間をゲーム化する。

* **2-1. エリア到着:** ナビゲーションはサークル範囲までで止まり、その先は自力探索になる。
* **2-2. 都市の余白の観察:** ガードレールの裏、配電盤の陰、看板の隙間など、普段見過ごす場所に視線が向く。
* **2-3. 不確かさのスリル:** 「本当にあるのか」「もうなくなったのでは」という疑心暗鬼自体が緊張感になる。
* **2-4. 文脈ある発見:** ユーザーまたはアンバサダー設置であれば、「なぜここに置いたか」という場所固有の意味も発見体験に重なる。

### フェーズ3：発見と刻印（Tagging）
**目的:** 発見を単なる通過ログではなく、誇れる記録に変える。

* **3-1. NFC主体・QR補助の読み取り:** 基本導線は NFC タグに埋めた URL を読み取って既存 Web を起動する方式とし、QR は NFC 非対応端末や補助ケースで使う。これにより、物理発見がデジタル記録へ接続される。
* **3-2. 劇的なリザルト演出:** 最初の発見者には「FTF達成！」バッジを付与し、既発見スポットでは「あなたは #42 番目の発見者です」のように発見順を表示する。
* **3-3. 鮮度の更新:** 読み取った瞬間に「Last Seen: Just now」へ更新され、スポットの存在感が次の探索者へ引き継がれる。
* **3-4. デジタル・タギング:** 発見者は短いストリートネームや一言コメント、写真や動画を残せる。
* **3-5. 設置者ステータスの表示:** そのフィギュアが「運営」「一般ユーザー」「アンバサダー」の誰による設置かを表示する。
* **3-6. デジタルウォール化:** キャラクターページには発見者のタグが蓄積し、デジタル上の壁のようにコミュニティ熱量が可視化される。

### フェーズ4：共鳴と次なる謎（Unlock）
**目的:** 1回の発見で終わらせず、コレクション欲と考察欲を次の行動につなげる。

* **4-1. コレクション確認:** マイページで、発見したキャラクター、発見順、FTFバッジをコレクションとして眺められる。
* **4-2. 組み合わせアンロック:** 特定のキャラクター同士を発見すると、「2つの存在が共鳴した」といった演出とともに、隠しスポットや新たなサークルが出現する。
* **4-3. 考察と共有:** ユーザーは「次に何が出るのか」「この組み合わせに意味があるのか」をSNSで語りたくなる。

### フェーズ5：消失・休眠と再覚醒（Sleeping & Awakening）
**目的:** 物理世界ならではの損失を弱点ではなく、世界観と収益導線に転換する。

* **5-1. 自動休眠:** 30日間読み取りがなければ、通常マップからスポットを消し、過疎感を防ぐ。
* **5-2. Legend表示:** 「Show Legends」をONにしたときだけ、過去の休眠スポットを金色のサークルとして表示し、都市の記憶として残す。
* **5-3. 物理キーによる再覚醒:** 物理キー用フィギュアを持つユーザーだけが、Legendの正確な位置を見られる。
* **5-4. 奇跡の再発見:** 現地に実物が残っていれば、再びActiveとして覚醒し、伝説的な発見体験になる。

### 3-7. リリース段階との対応

この5フェーズはプレイヤー体験の流れであり、事業のリリース段階とは切り分けて考える。  
実運用では、ニッチマーケ戦略に合わせて次の順で開放する。

1. **Phase 1a: 重点区での基礎ループ検証**  
   運営設置用フィギュア、曖昧サークル、Last Seen、ティッカー、NFC主体の発見記録までを先に成立させる。

2. **Phase 1b: 重点区内の密度拡張**  
   同一重点区内でスポット密度と発見記録を増やし、隣接区へ広げても探索価値が保てるかを確認する。

3. **Phase 2: 体験拡張としての収益接続**  
   物理キー用フィギュアや記念商品を追加し、休眠スポットの再覚醒やアーカイブ価値を課金導線へ接続する。

4. **Phase 3: 設置権の限定開放**  
   設置審査基準と運用体制が固まってから、招待制または審査制で設置用フィギュアを開放する。

## 4. 設置とコミュニティ拡張

### 4-1. 一般ユーザーによる設置

* ユーザーは設置用フィギュアを購入し、運用対象エリア内の余白候補を申請する。
* Webマップ / ゲームサイトから、設置場所と「なぜここに置いたか」という文脈を申請する。
* 運営はブランド毀損リスクだけでなく、安全性、私有地 / 公共交通インフラ該当性、迷惑行為誘発性を確認し、承認後にマップへ反映する。
* 初期段階では重点区内または運営指定エリアに限定し、全国一律には開放しない。

### 4-2. アンバサダー制度

* 設置の質や文脈の強さが高いユーザーを、運営が招待制でアンバサダーへ昇格させる。
* ポイント制ではなく招待制にすることで、称号の希少性と文化的権威を保つ。
* アンバサダー設置スポットは発見時に特別な表示が付き、コミュニティ内の憧れや尊敬を生む。

### 4-3. ユーザーの行動カテゴリと役割変化

この仕組みにより、ユーザーは細分化された行動ステータスを持ちながら、外形上は3つの行動カテゴリに整理できる。  
ゲームフロー設計では 6 つの細分化ステータスを定義し、サイトや参加導線の説明では中程度のカテゴリとして束ねて扱う。

#### 行動カテゴリ

1. **探す人**  
   手がかりを見て、街の中から AhhHum の痕跡を探す人

2. **残す人**  
   見つけた事実を記録し、発見を積み重ねていく人

3. **広げる人**  
   自分の文脈を都市に置き、世界観を拡張していく人

#### 細分化ステータス

1. **観測者:** マップやティッカーを見て、街で何が起きているかを観測する人
2. **探索者:** 手がかりを頼りに現地へ探しに行く人
3. **発見者:** 実際に見つけて NFC / QR で発見を記録する人
4. **収集者:** 発見順、FTF、組み合わせ、履歴を追いながら継続参加する人
5. **設置者:** 自分の文脈を都市へ埋め込み、設置によって参加する人
6. **共創者:** 世界観や文化の拡張に責任を持ち、継続的に関与する人

#### カテゴリとの対応関係

- **探す人:** 観測者、探索者
- **残す人:** 発見者、収集者
- **広げる人:** 設置者、共創者

この整理により、内部設計では行動変化を細かく追いながら、外部説明では `探す -> 残す -> 広げる` の3段階で参加方法を分かりやすく伝えられる。

## 5. UI/画面設計の要点

| 画面名 | 設計方針 |
| :--- | :--- |
| **マップ画面** | ピンではなくサークル表示。ティッカー、Last Seen、FTF状態、Legend切替を持つ。 |
| **スポット詳細** | 正確な住所は出さず、ヒント、シルエット、キャラクター視点写真、設置文脈を見せる。重点区までは伝えても即特定できる情報は出しすぎない。 |
| **読み取り後画面** | FTF達成、発見順、設置者属性、デジタルタグ投稿を強く演出する。 |
| **キャラクターページ** | 情報ページではなく、発見者のタグが蓄積する「デジタル・グラフィティ・ウォール」として機能させる。 |
| **マイページ** | 発見履歴一覧ではなく、シリアル番号、FTF、共鳴解除状況を見せるコレクションUIにする。 |

## 6. 結論

この統合ゲームフローにより、AhhHum ブランドは以下を同時に実現する。

1. **探索の面白さ:** 正解を隠すことで、都市を読むゲームになる。
2. **発見の誇り:** FTF、発見順、デジタルタグで「見つけた価値」が残る。
3. **継続の仕組み:** 共鳴アンロックと休眠・再覚醒により、1回で終わらない。
4. **共創の拡張:** 運営だけでなく、ユーザーとアンバサダーが世界を増殖させる。
5. **事業の持続性:** 物理キーと設置用フィギュアによって、参加体験と収益が接続される。

つまりこのゲームフローは、AhhHum を単なる「フィギュアの場所共有マップ」に留めず、**都市を舞台にした探索ゲーム、デジタル上のタギング文化、そしてユーザー共創型IP**へ接続する。
