# クラスタ範囲表示とパルスアニメーション統合仕様

## 概要

クラスタ範囲表示機能（点線サークル）をパルスアニメーションに統合し、パルスアニメーションのサイズをクラスタ範囲に合わせて可変にする仕様です。

## 現状の実装

### 更新可能性のパルスアニメーション
- **表示条件**: 更新可能なスポットがあるクラスタのみ
- **サイズ計算**: 更新可能なスポットの中心点と最大距離を計算
- **サイズ制限**: 最小40px、最大200px
- **動的更新**: ズームレベルに応じてサイズを変更

### クラスタ範囲表示（点線サークル）
- **表示条件**: クラスタをホバーまたはクリックした時
- **サイズ計算**: `calculateClusterRange()`を使用
  - クラスタ内の全スポットの中心点を計算
  - 中心点から最も遠いスポットまでの距離を計算
  - スポット間の最大距離も考慮
  - 50%の余裕を持たせる（`baseRadius * 1.5`）
  - 最小半径: 10メートル
- **表示方法**: Mapboxのfillレイヤーとlineレイヤー（点線）

## 統合後の仕様

### 基本方針
1. **点線サークルを削除**: `showClusterRange()`関数と関連レイヤーを削除
2. **パルスアニメーションを拡張**: 更新可能性の有無に関係なく、すべてのクラスタで表示
3. **サイズをクラスタ範囲に合わせる**: `calculateClusterRange()`で計算された半径を使用
4. **表示タイミングを維持**: ホバー時とクリック時の動作を維持

### 新しいパルスアニメーションの仕様

#### 表示条件と優先順位

クラスタの状態に応じて、以下の4つのパターンでパルスアニメーションを表示します：

| 優先順位 | 条件 | パルスカラー | 波紋の強弱/大小 | 説明 |
|---------|------|------------|---------------|------|
| 1（最優先） | 更新可能性 + ホットスポット（両方） | 赤 (#ef4444) | **強**（大きい波紋、速い周期） | 両方の条件を満たす最重要クラスタ |
| 2 | 更新可能なスポットのみ（ホットスポットではない） | オレンジ/金色 (#f59e0b) | **中**（中程度の波紋、標準周期） | 更新可能だがホットスポットではない |
| 3 | ホットスポットのみ（更新可能性ではない） | 青 (#3b82f6) | **中**（中程度の波紋、標準周期） | ホットスポットだが更新可能ではない |
| 4 | 更新可能でもホットスポットでもない | **パルスなし** | - | デフォルト表示（通常のクラスタマーカーのみ） |

**判定ロジック**:
1. クラスタ内に更新可能なスポットがあるか確認
2. クラスタがホットスポットかどうか確認（`recent_count >= MIN_COUNT`）
3. 両方の条件を満たす場合 → 優先順位1（赤、強）
4. 更新可能性のみの場合 → 優先順位2（オレンジ/金色、中）
5. ホットスポットのみの場合 → 優先順位3（青、中）
6. どちらでもない場合 → パルスなし

#### 表示タイミング
- **ホバー時**: 一時表示（マウスを離すと消える）
- **クリック時**: 固定表示（地図を移動/ズームしても表示を維持）

#### サイズ計算
- **関数**: `calculateClusterRange()`を使用（既存の関数を再利用）
- **計算方法**:
  1. クラスタ内の全スポットの中心点を計算
  2. 中心点から最も遠いスポットまでの距離を計算
  3. スポット間の最大距離も考慮
  4. より大きい方を使用し、50%の余裕を持たせる
  5. 最小半径: 10メートル
  6. 最終半径: `max(10m, max(maxDistance, maxSpotDistance / 2) * 1.5)`
- **ピクセル変換**: ズームレベルに応じてメートルをピクセルに変換
  - 緯度1度あたりのメートル数: 約111,000m
  - ズームレベルに応じたピクセル数: `256 * 2^zoom`
  - ピクセル半径: `(radiusMeters / metersPerDegree) * pixelsPerDegree`
- **サイズ制限**: 
  - 最小サイズ: 40px（視認性を確保）
  - 最大サイズ: 制限なし、または適切な上限を設定

#### 色とスタイル（条件別）

##### 1. 更新可能性 + ホットスポット（最優先）
- **色**: 赤 (#ef4444)
- **ボーダー**: 4px solid（太め）
- **背景**: transparent
- **波紋の強弱**: **強**
  - **サイズ**: 大きい（通常の1.2倍）
  - **周期**: 1.5秒（速い）
  - **拡大率**: scale(0.8) → scale(2.0)（大きい波紋）
  - **透明度変化**: 0.9 → 0.3 → 0（濃い）

##### 2. 更新可能なスポットのみ
- **色**: オレンジ/金色 (#f59e0b)
- **ボーダー**: 3px solid（標準）
- **背景**: transparent
- **波紋の強弱**: **中**
  - **サイズ**: 標準
  - **周期**: 2秒（標準）
  - **拡大率**: scale(0.8) → scale(1.5)（標準）
  - **透明度変化**: 0.8 → 0.4 → 0（標準）

##### 3. ホットスポットのみ
- **色**: 青 (#3b82f6)
- **ボーダー**: 3px solid（標準）
- **背景**: transparent
- **波紋の強弱**: **中**
  - **サイズ**: 標準
  - **周期**: 2秒（標準）
  - **拡大率**: scale(0.8) → scale(1.5)（標準）
  - **透明度変化**: 0.8 → 0.4 → 0（標準）

##### 4. 更新可能でもホットスポットでもない
- **パルスアニメーション**: 表示しない
- **表示**: 通常のクラスタマーカーのみ

#### アニメーション仕様（条件別）

##### 強（更新可能性 + ホットスポット）
- **アニメーション名**: `cluster-range-pulse-strong`
- **周期**: 1.5秒（速い）
- **イージング**: `ease-out`
- **繰り返し**: 無限 (`infinite`)
- **パルス数**: 3つ
- **遅延時間**: 0秒, 0.3秒, 0.6秒（短い間隔で連続的）
- **サイズ倍率**: 1.2倍（通常より大きい）
- **アニメーション**:
  - **開始**: scale(0.8), opacity 0.9, border-width 4px
  - **中間（50%）**: opacity 0.3
  - **終了**: scale(2.0), opacity 0, border-width 1px
- **視覚的効果**: 大きく、速く、濃い波紋で注意を引く

##### 中（更新可能性のみ / ホットスポットのみ）
- **アニメーション名**: `cluster-range-pulse-medium`
- **周期**: 2秒（標準）
- **イージング**: `ease-out`
- **繰り返し**: 無限 (`infinite`)
- **パルス数**: 3つ
- **遅延時間**: 0秒, 0.5秒, 1秒（標準間隔）
- **サイズ倍率**: 1.0倍（標準）
- **アニメーション**:
  - **開始**: scale(0.8), opacity 0.8, border-width 3px
  - **中間（50%）**: opacity 0.4
  - **終了**: scale(1.5), opacity 0, border-width 1px
- **視覚的効果**: 標準的な波紋で情報を伝える

#### 動的サイズ更新
- **更新タイミング**: 
  - ズーム変更時: `map.on("zoom", updateSize)`
  - 地図移動完了時: `map.on("moveend", updateSize)`
- **更新方法**: すべてのパルス円のサイズを更新

### 実装の変更点

#### 1. `createUpdatableClusterMarker`関数の拡張
- **関数名**: `createClusterRangePulseMarker`に変更（または既存の関数を拡張）
- **パラメータ追加**:
  - `clusterSpots: Spot[]` - クラスタ内の全スポット
  - `isUpdatable: boolean` - 更新可能性の有無
  - `isHotSpot: boolean` - ホットスポットの有無
  - `intensity: "strong" | "medium" | "none"` - 波紋の強弱（条件に応じて自動決定）
- **サイズ計算**: `calculateClusterRange()`を使用（クラスタ内の全スポットを対象）
- **色とスタイル**: 条件に応じて自動決定
  - `isUpdatable && isHotSpot` → 赤 (#ef4444)、強（周期1.5秒、拡大率2.0、サイズ1.2倍）
  - `isUpdatable && !isHotSpot` → オレンジ/金色 (#f59e0b)、中（周期2秒、拡大率1.5、サイズ1.0倍）
  - `!isUpdatable && isHotSpot` → 青 (#3b82f6)、中（周期2秒、拡大率1.5、サイズ1.0倍）
  - `!isUpdatable && !isHotSpot` → パルスなし（関数を呼び出さない）

#### 2. `showClusterRange`関数の削除
- 点線サークル表示の関数を削除
- 関連するレイヤー（`cluster-range-circle-layer`）も削除

#### 3. クラスタクリック/ホバー時の処理変更
- **ホバー時**: パルスアニメーションを一時表示
  - 更新可能性とホットスポットの状態を確認
  - 条件に応じたパルスアニメーションを表示
- **クリック時**: パルスアニメーションを固定表示
  - 更新可能性とホットスポットの状態を確認
  - 条件に応じたパルスアニメーションを表示
- **表示条件**: 
  - 更新可能性またはホットスポットのいずれかがある場合のみ表示
  - 両方ともない場合は、パルスアニメーションを表示しない（通常のクラスタマーカーのみ）

#### 4. マーカー管理
- **更新可能性があるクラスタ**: オレンジ/金色のパルス
- **更新可能性がないクラスタ**: 青いパルス
- **両方のマーカーを管理**: `clusterRangePulseMarkersRef`で管理

### 表示の優先順位

#### z-index順序
1. **更新可能性のパルス**（オレンジ/金色）: 1001（最前面）
2. **通常のクラスタ範囲パルス**（青）: 1000
3. **ホットスポットパルス**: 999（背景）

#### 視覚的優先度
1. **更新可能性のパルス**（最優先）
   - ユーザーがアクションを起こすべき重要な情報
   - 常に最前面に表示

2. **通常のクラスタ範囲パルス**（中優先度）
   - クラスタの範囲を示す補助的な情報
   - 更新可能性のパルスの背景として表示

3. **ホットスポットパルス**（低優先度）
   - エリアの活発さを示す補助的な情報
   - 背景として表示

### 削除する機能

#### 1. 点線サークル表示
- `showClusterRange()`関数
- `clearClusterRange()`関数（パルスアニメーション用に再実装）
- `clusterRangeCircleRef`
- `cluster-range-circle`ソース
- `cluster-range-circle-layer`レイヤー
- `cluster-range-circle-layer-outline`レイヤー

#### 2. スポットハイライト（オプション）
- `highlightClusterSpots()`関数
- `cluster-spots-highlight`レイヤー
- **注意**: スポットハイライトは残すか削除するか、要検討

### 保持する機能

#### 1. クラスタ範囲計算
- `calculateClusterRange()`関数 - パルスアニメーションのサイズ計算に使用
- `createCirclePolygon()`関数 - 削除（使用されなくなる）

#### 2. 選択状態の管理
- `selectedClusterSpotsRef` - クリック時に選択されたクラスタを保持
- 地図移動/ズーム時の再描画処理

### 実装の流れ

1. **新しい関数を作成**: `createClusterRangePulseMarker()`
   - `calculateClusterRange()`を使用してサイズを計算
   - 更新可能性の有無に応じて色を変更
   - ズームレベルに応じてサイズを動的に更新

2. **クラスタクリック/ホバー時の処理を変更**
   - `showClusterRange()`の代わりに`createClusterRangePulseMarker()`を呼び出し
   - 更新可能性の有無に関係なく、すべてのクラスタでパルスアニメーションを表示

3. **点線サークル関連のコードを削除**
   - `showClusterRange()`関数
   - `clearClusterRange()`関数（パルスアニメーション用に再実装）
   - 関連するレイヤーとソース

4. **マーカー管理を更新**
   - `clusterRangePulseMarkersRef`を追加
   - ホバー時とクリック時のマーカー管理を実装

### メリット

1. **視覚的な統一**: パルスアニメーションでクラスタ範囲を表示することで、視覚的に統一される
2. **パフォーマンス向上**: MapboxレイヤーではなくHTMLマーカーを使用することで、パフォーマンスが向上する可能性
3. **実装の簡素化**: 点線サークルとパルスアニメーションを統合することで、コードが簡素化される
4. **柔軟性**: パルスアニメーションのスタイルを変更することで、より柔軟な表現が可能

### 注意点

1. **更新可能性のパルスとの区別**: 更新可能性のパルスと通常のクラスタ範囲パルスを視覚的に区別する必要がある
2. **パフォーマンス**: 多くのクラスタでパルスアニメーションを表示すると、パフォーマンスに影響する可能性がある
3. **視認性**: パルスアニメーションが点線サークルよりも視認性が低い可能性がある
