# Figuraffiti フェーズ1 実装メモ

要件定義「11. まとめ：フェーズ1で実装すべき最小セット」に沿って実装した内容のメモです。

## 実装済み

### 1. QR/NFCの飛び先（キャラクターページ）
- **DB**: `supabase/migrations/20250307000000_add_characters.sql` で `characters` テーブルを追加。`spots.character_id` で発見報告とキャラを紐づけ。
- **API**: `GET /api/characters`（一覧）、`GET /api/characters/[slug]`（1件＋発見ログ）。
- **ページ**: `/character/[slug]` — キャラ名・ストーリー・画像・発見ログを表示。スマホ向けレイアウト。
- **シード**: サンプルキャラ「パイロット」（slug: `pilot`）をマイグレーションで投入。QRの飛び先は `https://あなたのドメイン/character/pilot` のように設定可能。

### 2. オンラインマップ
- 既存の `app/mapping/page.tsx` と `components/map/map-view.tsx` をそのまま転用。文言のみ「Figuraffiti／発見報告」に変更済み。

### 3. 発見報告の投稿
- 既存の `app/submit/page.tsx` と `components/form/submit-form-bottom-sheet.tsx` を転用。ラベルを「技名」→「作品名・キャラ名」に変更。

### 4. 承認管理
- 既存の `app/admin/page.tsx` と API（approve / reject / pending）を転用。UI文言を「承認待ちの発見報告」に統一。

### 5. 認証・プロフィール
- 既存の `contexts/auth-context`、`app/profile/page.tsx`、`app/api/profile/*` を転用。プロフィールの「マイ投稿」はそのまま利用。

### 6. SNS埋め込み
- 既存の `app/api/instagram/oembed/route.ts` を転用。User-Agent を「Figuraffiti/1.0」に変更。TikTok は既存のメディアソース（`spot_media.source`）に含まれており、埋め込みはスポット詳細等で利用可能。

## ブランド・文言の統一

- **layout.tsx**: タイトル・description・OGP を Figuraffiti 用に変更。
- **constants.ts**: エラー／成功／UIテキストを「記録」→「発見報告」、「技名」→「作品名・キャラ名」等に変更。
- **site-header.tsx**: ロゴ「Skateright」→「Figuraffiti」、「遊び方」→「探し方」。
- **site-footer.tsx**: コピーとリンクを Figuraffiti 用に変更。
- **how-it-works.tsx / how-it-works/page.tsx**: 2ステップの説明を発見報告用に変更。
- **guidelines/page.tsx**: ガイドライン本文を Figuraffiti 用に差し替え。

## アーカイブ・無効化

- **\_archive/README.md**: 削除・無効化した機能（Discord、スキルレベル等）のメモを記載。
- **Discord**: `app/api/discord/interactions/route.ts` は従来どおり 410 スタブのまま。Discord 通知呼び出しは既存コードで削除済み。

## 次の作業（運用側）

1. **マイグレーション実行**: Supabase で `supabase/migrations/20250307000000_add_characters.sql` を実行し、`characters` テーブルと `spots.character_id` を作成する。
2. **QRコード**: フィギュアに貼るQRのURLを `https://（本番ドメイン）/character/pilot` のように設定する。
3. **発見ログをキャラに紐づける**: 発見報告を承認する際、そのスポットをどのキャラに紐づけるかは現状、DBの `spots.character_id` を手動で更新する必要がある。フェーズ2で管理画面から「キャラクター選択」を追加可能。
