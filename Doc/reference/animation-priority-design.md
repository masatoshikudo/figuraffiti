# アニメーション優先順位の設計

## 概要

地図上で複数のアニメーションが同時に表示される場合の優先順位を定義します。

## アニメーションの種類

1. **更新可能なスポットのアニメーション（パルス）**
   - 色: オレンジ/金色 (#f59e0b)
   - アニメーション: パルス（拡大縮小）
   - 意味: ユーザーがアクションを起こすべき重要な情報

2. **高難易度スポットのアニメーション（グロー）**
   - 色: 紫 (#8b5cf6)
   - アニメーション: グロー（光るエフェクト）
   - 意味: 上級技の記録を示す

3. **ホットスポットのアニメーション（パルス）**
   - 色: 青/緑/赤（強度に応じて）
   - アニメーション: パルス（拡大縮小）
   - 意味: エリアの活発さを示す

## 優先順位

### 優先順位1: 更新可能なスポット（最優先）
- **理由**: ユーザーがアクションを起こすべき重要な情報
- **視覚的優先度**: 最高
- **z-index**: 1001（最前面）

### 優先順位2: 高難易度スポット
- **理由**: 上級技の記録を示す重要な情報
- **視覚的優先度**: 高
- **z-index**: 1000

### 優先順位3: ホットスポット（背景）
- **理由**: エリアの活発さを示す補助的な情報
- **視覚的優先度**: 中
- **z-index**: 999（背景）

## 実装方針

### Mapboxレイヤーの順序
1. ホットスポットマーカー（HTMLマーカー、z-index: 999）
2. クラスタレイヤー
3. 個別スポットレイヤー（通常）
4. 高難易度スポット（レイヤー内で自動的に処理）
5. 更新可能なスポット（レイヤー内で自動的に処理）
6. ホバー中のスポット/クラスタ（最前面）

### アニメーションの重複時の動作
- 更新可能なスポットとホットスポットが重なる場合:
  - 更新可能なスポットのアニメーションが前面に表示される
  - ホットスポットのアニメーションは背景として表示される

- 高難易度スポットとホットスポットが重なる場合:
  - 高難易度スポットのアニメーションが前面に表示される
  - ホットスポットのアニメーションは背景として表示される

## 視覚的な区別

### 更新可能なスポット
- パルスアニメーション（1.5秒周期）
- オレンジ/金色で目立つ
- サイズが大きめ（28px）

### 高難易度スポット
- グローエフェクト（1.5秒周期）
- 紫色で目立つ
- サイズが大きめ（26px）

### ホットスポット
- パルスアニメーション（2秒周期）
- 強度に応じた色（青/緑/赤）
- 大きな円形マーカー（40-60px）

## 注意事項

- アニメーションが多すぎると視覚的に混乱する可能性がある
- ズームレベルに応じて、ホットスポットの表示を制御することを推奨
- 更新可能なスポットと高難易度スポットが重なる場合は、更新可能なスポットを優先
