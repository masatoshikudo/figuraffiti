# 提案B「最初の指令」インタラクティブ・チュートリアル 実装計画

## 概要

提案Bは、新規ユーザーにゲーム風のチュートリアルを通じて、サービスのコアな価値（記録の発見と更新可能性の理解）を能動的に学ばせる機能です。

## 実装目標

1. **ミッション開始**: サイト訪問時、ユーザーの現在位置を取得し、その地域名（例：「渋谷」）に合わせて「最初の指令：[地域名]の『更新可能』な記録を見つけ出せ」といったポップアップを表示
2. **探索と発見**: ユーザーが地図を操作すると、ヒントを表示し、目的のスポットへ誘導。更新可能なスポットをアニメーションで目立たせる
3. **目標の理解**: 正しいピンをクリックすると、対話形式で次のアクションを促す
4. **状態の保存**: チュートリアル完了状態を保存し、次回以降は表示しない

## 技術要件

### 1. チュートリアルエンジン（状態管理）

**実装内容:**
- チュートリアルのステップを管理する状態管理システム
- 各ステップでのユーザーアクションを検知
- 次のステップへの遷移ロジック

**必要な機能:**
- ステップ定義（enumまたは定数）
- 現在のステップ状態管理
- ステップ遷移ロジック
- アクション検知（地図操作、ピンクリックなど）

### 2. 位置情報取得と動的エリア選択機能

**実装内容:**
- ユーザーの現在位置を取得（Geolocation API）
- 位置情報から地名（区名など）を取得（逆ジオコーディング）
- 取得した地名に基づいてエリアを動的に決定
- **地図を自動的に目標エリアにズーム**（移動は自由、ユーザーはどこにでも移動可能）
- **ヒントとアニメーションによる誘導**: エリア限定は行わず、視覚的な誘導で目標スポットへ導く

**必要な機能:**
- Geolocation APIを使用した現在位置の取得
- Mapbox Geocoding APIまたはGoogle Places APIを使用した逆ジオコーディング
- 地名からエリアの中心点とズームレベルを決定するロジック
- エリアの中心点とズームレベルの定義（地図の自動ズーム用）
- 地図の自動ズーム機能（`flyTo`を使用）

**設計方針:**
エリア限定機能は実装せず、以下の方法で誘導します：
- ✅ **ヒントメッセージ**: 「このエリアを探索してみて」などのメッセージで誘導
- ✅ **アニメーション強調**: 更新可能なスポットのアニメーションで視覚的に誘導
- ✅ **自動ズーム**: 地図を自動的に目標エリアにズーム（ただし移動は自由）
- ✅ **ユーザーの自由度**: ユーザーはどこにでも移動可能で、自由度が高い

### 3. ヒント表示機能とアニメーション強調

**実装内容:**
- ユーザーの操作に応じて動的にヒントを表示
- オーバーレイUIでヒントメッセージを表示
- **更新可能なスポットをアニメーションで目立たせる**（パルス、グロー、点滅など）
- 目的のスポットへの視覚的な誘導（アニメーション、ハイライトなど）

**必要な機能:**
- ヒントメッセージの管理
- オーバーレイUIコンポーネント
- **更新可能なスポットのアニメーション効果**（CSSアニメーション、Mapboxマーカーの動的スタイル変更）
- スポットのハイライト表示
- アニメーション効果（パルス、グロー、点滅など）

### 4. インタラクション検知

**実装内容:**
- ユーザーが正しいピン（更新可能な記録）をクリックしたことを検知
- 間違ったピンをクリックした場合のフィードバック
- クリックイベントのハンドリング

**必要な機能:**
- スポットの更新可能性判定（既存の`canUpdateRecord`を活用）
- クリックイベントの検知と判定
- フィードバック表示（成功/失敗）

### 5. 対話形式UI

**実装内容:**
- ユーザーに語りかけるような対話形式のメッセージ
- ポップアップやモーダルでメッセージを表示
- 次のアクションを促すボタンやリンク

**必要な機能:**
- 対話形式UIコンポーネント
- メッセージテンプレート
- アクションボタンの実装

### 6. 状態の保存

**実装内容:**
- チュートリアル完了状態をlocalStorageに保存
- 次回訪問時はチュートリアルをスキップ
- オプション：ユーザーがチュートリアルを再開できる機能

**必要な機能:**
- localStorageへの保存/読み込み
- 完了状態のチェック
- 再開機能（オプション）

## 技術的な考慮事項

### 1. 位置情報取得とエリア定義

**位置情報取得:**
- Geolocation APIを使用してユーザーの現在位置を取得
- 位置情報の取得に失敗した場合は、デフォルトエリア（例：東京）を使用

**逆ジオコーディング:**
- Mapbox Geocoding APIまたはGoogle Places APIを使用して座標から地名を取得
- 区名や市名を抽出してメッセージに使用

**エリア定義:**
主要エリアの中心点とズームレベルを定義します（地図の自動ズーム用）。例：
```typescript
export const AREA_CENTERS = {
  shibuya: {
    name: "渋谷",
    center: [139.70, 35.66], // [経度, 緯度]
    zoom: 14,
  },
  shinjuku: {
    name: "新宿",
    center: [139.70, 35.68],
    zoom: 14,
  },
  // その他の主要エリア...
}

// 地名からエリアをマッピング
export const AREA_NAME_MAPPING: Record<string, keyof typeof AREA_CENTERS> = {
  "渋谷区": "shibuya",
  "渋谷": "shibuya",
  "新宿区": "shinjuku",
  "新宿": "shinjuku",
  // その他のマッピング...
}
```

**注意**: エリア限定は行わないため、境界ボックスは不要です。中心点とズームレベルのみを定義します。

### 2. 更新可能性の判定

既存の`canUpdateRecord()`関数を活用しますが、チュートリアル用には以下の条件を満たす記録が必要です：
- 承認済み（`status === 'approved'`または未設定）
- より高難易度の技で更新可能

**注意**: エリア限定は行わないため、すべての更新可能な記録を対象とします。ただし、ユーザーの位置情報に近い記録を優先的に選択します。

### 3. 状態管理

React Context APIを使用してチュートリアルの状態を管理します。これにより、複数のコンポーネント間で状態を共有できます。

### 4. アニメーション強調

更新可能なスポットを目立たせるためのアニメーション効果：
- **パルスアニメーション**: マーカーが拡大縮小を繰り返す
- **グローエフェクト**: マーカーの周りに光るエフェクト
- **点滅アニメーション**: マーカーが点滅する
- **カラー強調**: 更新可能なスポットのマーカーを特別な色（例：金色、オレンジ）で表示

実装方法：
- CSS animationsを使用したマーカーのスタイル変更
- Mapboxマーカーの動的なスタイル更新
- 定期的な再レンダリングでアニメーション効果を実現

### 5. パフォーマンス

- 更新可能性の判定は事前に計算してキャッシュ
- 不要な再レンダリングを避けるためにメモ化を適用
- アニメーション効果は`requestAnimationFrame`を使用して最適化
- すべてのスポットを表示するため、パフォーマンスに注意（必要に応じてビューポート内のスポットのみを表示）

### 6. アクセシビリティ

- キーボード操作に対応
- スクリーンリーダー対応
- モバイルデバイスでの操作性を考慮
- 位置情報の取得はユーザーの許可が必要（プライバシー配慮）

## データ構造

### チュートリアルステップ定義

```typescript
export enum TutorialStep {
  NOT_STARTED = 'not_started',
  MISSION_START = 'mission_start',
  EXPLORING = 'exploring',
  FOUND_TARGET = 'found_target',
  COMPLETED = 'completed',
}

export interface TutorialConfig {
  area: {
    name: string // エリア名（例：「渋谷」）
    center: [number, number] // 地図の自動ズーム用の中心点
    zoom: number // 地図の自動ズーム用のズームレベル
  }
  targetSpot?: Spot
  messages: {
    [key in TutorialStep]: {
      title: string
      message: string // エリア名を動的に挿入可能（例：「{areaName}の『更新可能』な記録を見つけ出せ」）
      hint?: string
      action?: string
    }
  }
}
```

### チュートリアル状態

```typescript
export interface TutorialState {
  step: TutorialStep
  isActive: boolean
  completed: boolean
  targetSpotId?: string
  userLocation?: {
    lat: number
    lng: number
  }
  areaName?: string // 取得したエリア名（例：「渋谷」）
  areaConfig?: TutorialConfig['area'] // 決定されたエリア設定
}
```

5. **目標スポットの選定**: 更新可能な記録がない場合の処理

7. **デザイン**: UIコンポーネントのデザイン方向性の確認

**設計方針（確定）:**
- ✅ **エリア限定なし**: ユーザーは自由に地図を移動可能
- ✅ **自動ズーム**: チュートリアル開始時に目標エリアに自動ズーム
- ✅ **視覚的誘導**: ヒントメッセージとアニメーション強調で目標スポットへ誘導

承認後、実装を開始いたします。
