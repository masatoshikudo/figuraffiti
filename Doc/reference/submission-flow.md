# 記録を残すフロー

## フロー概要

1. **サインアップ・ログイン**
   - ユーザーがサインアップまたはログイン
   - 認証が必要（記録送信には認証必須）

2. **仮ピンを立てる**
   - `/mapping` ページで地図をクリック
   - 仮ピンが表示され、記録フォームが開く

3. **記録情報を入力**
   - **トリック名**: 技名を入力（例: Ollie, Kickflip）
   - **SNSのリンク**: Instagram、YouTubeなどのURLを入力

4. **記録を送信**
   - 「記録を残す」ボタンを押下
   - データベースに保存される

## 送信後の挙動

### 信頼ユーザーでない場合（一般ユーザー）

1. **ステータス**: `pending`（承認待ち）で保存される
2. **レスポンス**: 「投稿を受け付けました。承認後に公開されます。」というメッセージが返される
3. **表示**: 地図には表示されない（承認待ちのため）
4. **通知**:
   - ✅ **オーナー（管理者）へのメール通知**: 新しい投稿が承認待ちであることを通知
   - ✅ **ユーザーへのメール通知**: 承認/却下時に通知（承認時または却下時）

### 信頼ユーザーの場合

1. **ステータス**: `approved`（承認済み）で保存される
2. **レスポンス**: 作成されたスポットが返される
3. **表示**: すぐに地図に表示される（自動承認のため）
4. **通知**: なし（自動承認のため）

## 承認プロセス

1. **管理者が `/admin` ページにアクセス**
2. **承認待ちリストを確認**
3. **各スポットに対して承認/却下を実行**
   - **承認**: ステータスを `approved` に変更 → 地図に表示される
   - **却下**: ステータスを `rejected` に変更 → 地図に表示されない

## メール通知のタイミング

### 投稿時（新規投稿が承認待ちの場合）

- **送信先**: 管理者（`admin_users` テーブルに登録されているユーザー）
- **内容**: 
  - 件名: 「新しい記録が承認待ちです」
  - 本文: スポット名、投稿者、投稿日時、承認ページへのリンク

### 承認時

- **送信先**: 投稿者（`submitted_by` が設定されている場合）
- **内容**: 
  - 件名: 「スポットが承認されました」
  - 本文: スポット名、承認日時

### 却下時

- **送信先**: 投稿者（`submitted_by` が設定されている場合）
- **内容**: 
  - 件名: 「スポットが却下されました」
  - 本文: スポット名、却下理由、却下日時

## テストフロー

1. **サインアップ**: 新しいアカウントを作成
2. **ログイン**: 作成したアカウントでログイン
3. **仮ピンを立てる**: `/mapping` ページで地図をクリック
4. **記録情報を入力**:
   - トリック名: 「Ollie」
   - SNSのリンク: 「https://www.instagram.com/p/example」
5. **記録を送信**: 「記録を残す」ボタンを押下
6. **確認**:
   - フロントエンド: 「投稿を受け付けました。承認後に公開されます。」というメッセージが表示される
   - メール: 管理者にメール通知が送信される（実装予定）
   - データベース: `status='pending'` で保存される
7. **管理者が承認**:
   - `/admin` ページで承認待ちリストを確認
   - 承認ボタンを押下
   - ユーザーにメール通知が送信される
   - 地図に表示される
