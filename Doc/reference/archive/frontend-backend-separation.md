# フロントエンドとバックエンドの分離設計

## 概要

フロントエンド（一般ユーザー向け）とバックエンド（管理機能）を明確に分離し、それぞれ独立したページとして実装します。

---

## ページ構成

### フロントエンド（Frontend）

**対象ユーザー**: すべてのユーザー（認証不要のページも含む）

#### 公開ページ（認証不要）
- `/` - ホームページ
- `/list` - スポット一覧
- `/spot/[id]` - スポット詳細
- `/submit` - スポット提案フォーム
- `/mapping` - マッピングページ
- `/guidelines` - ガイドライン
- `/terms` - 利用規約

#### 認証が必要なページ
- `/profile` - プロフィールページ
  - マイ投稿タブ: 自分の投稿一覧
  - 設定タブ: アカウント設定、通知設定

**特徴:**
- ユーザー個人の情報と設定のみ
- 管理機能は含まない
- すべての認証済みユーザーがアクセス可能

---

### バックエンド（Backend / Admin）

**対象ユーザー**: 管理者（Admin）のみ

#### 管理機能専用ページ
- `/admin` - 管理画面（管理者専用）
  - 承認管理タブ: 承認待ちスポットの承認/却下
  - データ閲覧タブ: 全スポットの閲覧（読み取り専用）、統計情報
  - ユーザー管理タブ: 信頼ユーザーの追加/削除

**特徴:**
- 管理者の日常的な承認業務に特化
- 権限チェック必須（管理者以外は403エラー）
- フロントエンドとは完全に分離

**オーナー（Owner）について:**
- `/admin`ページは不要（オプション: 実装しない）
- Supabase管理画面で直接操作することを推奨
  - より効率的な操作が可能
  - より強力な機能が利用可能
  - 開発コストを削減

---

## アクセス制御

### フロントエンド（`/profile`）
- **認証チェック**: 必須
- **権限チェック**: 不要（すべての認証済みユーザーがアクセス可能）
- **リダイレクト**: 未認証の場合は認証ダイアログを表示

### バックエンド（`/admin`）
- **認証チェック**: 必須
- **権限チェック**: 必須（管理者/オーナーのみ）
- **リダイレクト**: 
  - 未認証の場合 → `/profile`または認証ダイアログ
  - 権限不足の場合 → `/profile`または403エラーページ

---

## ナビゲーション

### ヘッダーナビゲーション
- フロントエンドページへのリンクのみ
- ユーザーメニューに「管理画面」リンクを追加（管理者/オーナーのみ表示）

### ユーザーメニュー
```
- プロフィール (/profile) - すべての認証済みユーザー
- 管理画面 (/admin) - 管理者のみ表示（オーナーは表示しない）
- ログアウト
```

**オーナー向けの案内:**
- `/profile`ページに「Supabase管理画面へのリンク」を表示（オプション）
- または、オーナーにも`/admin`ページへのアクセスを許可（管理者と同じ機能）

### モバイルタブバー
- フロントエンドページへのリンクのみ
- 管理画面へのリンクは含まない（ヘッダーのユーザーメニューからアクセス）

---

## 実装方針

### 1. `/profile`ページの変更
- 管理機能タブを削除
- マイ投稿タブと設定タブのみ残す
- `AdminSectionTab`コンポーネントを削除

### 2. `/admin`ページの作成/更新
- 既存の`/admin`ページを更新
- 権限チェックを追加
- 管理者/オーナー向けのタブ構成を実装
- `AdminSectionTab`コンポーネントを再利用

### 3. 権限チェックの実装
- `/admin`ページで権限チェックを実行
- 権限不足の場合は適切なエラーメッセージまたはリダイレクト

### 4. ナビゲーションの更新
- ユーザーメニューに「管理画面」リンクを追加
- 管理者/オーナーのみ表示

---

## ファイル構成

### フロントエンド
```
app/
  profile/
    page.tsx                    # プロフィールページ（管理機能なし）
    components/
      my-submissions-tab.tsx   # マイ投稿タブ
      settings-tab.tsx         # 設定タブ
```

### バックエンド
```
app/
  admin/
    page.tsx                    # 管理画面（権限チェック必須）
    components/                 # 管理機能コンポーネント（必要に応じて）
```

### 共有コンポーネント
```
components/
  admin/
    admin-section-tab.tsx      # 管理機能タブ（/adminで使用）
    approval-management.tsx    # 承認管理コンポーネント
    data-view.tsx              # データ閲覧コンポーネント
    user-management.tsx        # ユーザー管理コンポーネント
    data-management.tsx        # データ管理コンポーネント（オーナーのみ）
    csv-import.tsx             # CSVインポート（オーナーのみ）
    admin-management.tsx       # 管理者管理（オーナーのみ）
```

## セキュリティ考慮事項

1. **フロントエンドの権限チェック**
   - UIの表示/非表示はUXのため
   - セキュリティはAPIルートで保証

2. **バックエンドの権限チェック**
   - ページレベルで権限チェックを実行
   - APIルートでも権限チェックを実行（二重チェック）
   - 権限不足の場合は403エラーを返す

3. **URLの直接アクセス**
   - `/admin`に直接アクセスした場合も権限チェックを実行
   - 権限不足の場合は適切なエラーメッセージを表示

## 移行計画

### フェーズ1: `/profile`ページの整理
1. `AdminSectionTab`コンポーネントを`/profile`から削除
2. マイ投稿タブと設定タブのみ残す
3. 権限チェックを削除（すべての認証済みユーザーがアクセス可能）

### フェーズ2: `/admin`ページの実装
1. 既存の`/admin`ページを確認・更新
2. 権限チェックを追加
3. `AdminSectionTab`コンポーネントを`/admin`で使用
4. 管理者/オーナー向けのタブ構成を実装

### フェーズ3: ナビゲーションの更新
1. ユーザーメニューに「管理画面」リンクを追加
2. 管理者/オーナーのみ表示する条件を追加

### フェーズ4: テスト
1. 一般ユーザーで`/admin`にアクセス → 403エラーまたはリダイレクト
2. 管理者で`/admin`にアクセス → 管理画面が表示される
3. オーナーで`/admin`にアクセス → すべてのタブが表示される
