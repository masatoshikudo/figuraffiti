# サイトマップ

## 公開ページ（認証不要）

### `/` - ホームページ
- **説明**: ランディングページ、地図表示
- **権限**: 誰でもアクセス可能
- **機能**: 
  - ヒーローセクション
  - 遊び方説明
  - クエスト紹介
  - スポットギャラリー（承認済みスポットのみ）
  - コミュニティミッション
  - ロードマップ
  - CTA

### `/list` - スポット一覧
- **説明**: 承認済みスポットの一覧表示
- **権限**: 誰でもアクセス可能
- **機能**:
  - フィルタリング（都道府県、年代、ソース）
  - 検索
  - ソート

### `/spot/[id]` - スポット詳細
- **説明**: 個別スポットの詳細ページ
- **権限**: 誰でもアクセス可能（承認済みスポットのみ）
- **機能**:
  - スポット情報の表示
  - メディア表示（カバー、記事、動画）
  - 地図表示
  - 関連情報

### `/submit` - スポット提案フォーム
- **説明**: 新しいスポットの投稿フォーム
- **権限**: 誰でもアクセス可能（認証不要）
- **機能**:
  - スポット情報の入力
  - メディアの追加
  - 位置情報の選択
  - 投稿（status='pending'で作成）

### `/mapping` - マッピングページ
- **説明**: 地図上でスポットを閲覧
- **権限**: 誰でもアクセス可能
- **機能**:
  - インタラクティブな地図表示
  - スポットのマーカー表示
  - フィルタリング

### `/guidelines` - ガイドライン
- **説明**: 投稿ガイドライン、利用規約
- **権限**: 誰でもアクセス可能
- **機能**: 静的コンテンツ

### `/terms` - 利用規約
- **説明**: 利用規約、プライバシーポリシー
- **権限**: 誰でもアクセス可能
- **機能**: 静的コンテンツ

---

## フロントエンド（一般ユーザー向けページ）

### `/profile` - プロフィールページ
- **説明**: ユーザーのプロフィールと個人設定
- **権限**: 認証済みユーザー
- **機能**:
  - **マイ投稿タブ**: 自分の投稿一覧（全ステータス: 承認待ち、承認済み、却下済み）
  - **設定タブ**: アカウント設定、通知設定
- **特徴**: フロントエンド専用ページ、管理機能は含まない

---

## バックエンド（管理機能専用ページ）

### `/admin` - 管理画面
- **説明**: 管理者・オーナー専用の管理機能
- **権限**: 管理者以上（認証必須）
- **アクセス制御**: 管理者/オーナー以外は403エラーまたはリダイレクト

#### 管理者（Admin）がアクセス可能な機能:
- **承認管理タブ**:
  - 承認待ちスポットの一覧
  - スポットの承認/却下
  - 却下理由の入力
- **データ閲覧タブ**:
  - 全スポットの閲覧（読み取り専用）
  - 統計情報（都道府県別、年代別、ソース別）
  - フィルタリング・検索
- **ユーザー管理タブ**:
  - 信頼ユーザーの一覧
  - 信頼ユーザーの追加/削除

#### オーナー（Owner）がアクセス可能な機能:
- 管理者の機能すべて
- **データ管理タブ**:
  - 全スポットの編集・削除
  - スポットの一括操作
- **CSVインポートタブ**:
  - CSVファイルからのスポットデータ一括インポート
- **管理者管理タブ**:
  - 管理者一覧
  - 管理者の追加/削除
  - 権限レベルの変更

## APIエンドポイント

### 公開API（認証不要）
- `GET /api/spots` - 承認済みスポットの取得
- `POST /api/spots` - スポットの投稿（status='pending'）

### 認証が必要なAPI
- `GET /api/spots/my-submissions` - 自分の投稿一覧
- `PUT /api/spots?id=xxx` - スポットの編集（自分の投稿のみ、またはオーナーのみ）
- `DELETE /api/spots?id=xxx` - スポットの削除（自分の投稿のみ、またはオーナーのみ）

### 管理者API（管理者以上）
- `GET /api/spots/pending` - 承認待ちスポットの取得
- `POST /api/spots/approve` - スポットの承認
- `POST /api/spots/reject` - スポットの却下
- `GET /api/admin/trusted-users` - 信頼ユーザー一覧
- `POST /api/admin/trusted-users` - 信頼ユーザーの追加
- `DELETE /api/admin/trusted-users/[userId]` - 信頼ユーザーの削除

### オーナーAPI（オーナーのみ）
- `GET /api/admin/admins` - 管理者一覧（将来実装）
- `POST /api/admin/admins` - 管理者の追加（将来実装）
- `DELETE /api/admin/admins/[userId]` - 管理者の削除（将来実装）
- `PUT /api/admin/admins/[userId]` - 権限レベルの変更（将来実装）

## 権限レベル別アクセス可能ページ

### 一般ユーザー（User）
**フロントエンドページ:**
- `/` ✅
- `/list` ✅
- `/spot/[id]` ✅（承認済みのみ）
- `/submit` ✅
- `/mapping` ✅
- `/guidelines` ✅
- `/terms` ✅
- `/profile` ✅（マイ投稿、設定のみ）

**バックエンドページ:**
- `/admin` ❌（アクセス不可、403エラー）

### 管理者（Admin）
**フロントエンドページ:**
- 一般ユーザーがアクセス可能なすべてのページ ✅
- `/profile` ✅（マイ投稿、設定のみ）

**バックエンドページ:**
- `/admin` ✅
  - 承認管理 ✅
  - データ閲覧（読み取り専用）✅
  - 統計情報 ✅
  - 信頼ユーザー管理 ✅

### オーナー（Owner）
**フロントエンドページ:**
- 管理者がアクセス可能なすべてのページ ✅
- `/profile` ✅（マイ投稿、設定のみ）

**バックエンドページ:**
- `/admin` ⚠️（オプション: 実装しない、または管理者と同じ機能を提供）

**Supabase管理画面:**
- 直接アクセス可能 ✅（推奨）
  - データベースの直接編集・削除
  - CSVインポート（SQL Editorで実行）
  - 管理者の追加/削除（`admin_users`テーブルを直接編集）
  - RLSポリシーの変更
  - 統計情報の確認（SQLクエリで実行）

## ページ遷移フロー

### スポット投稿フロー（フロントエンド）
1. `/submit` - フォーム入力
2. 投稿送信 → `status='pending'`で作成
3. `/profile` - 「マイ投稿」タブで確認（承認待ち）

### 承認フロー（バックエンド）
1. `/admin` - 管理画面にアクセス（管理者/オーナーのみ）
2. 「承認管理」タブで承認待ちスポットを確認
3. 承認/却下を実行
4. 投稿者にメール通知（実装済み）

### データ管理フロー（オーナー - Supabase管理画面）
1. Supabase管理画面にアクセス
2. Table Editorでデータを直接編集・削除
3. SQL EditorでCSVインポートや一括操作を実行
4. または、必要に応じて`/admin`ページを使用（オプション）
