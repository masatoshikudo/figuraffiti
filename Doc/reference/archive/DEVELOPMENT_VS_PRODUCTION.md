# 開発環境と本番環境のデータベースについて

## 現在の状況

### 開発環境
- **データベース**: ローカルSupabase（`http://127.0.0.1:54321`）
- **データ**: `supabase/seed.sql` のテストデータ（ダミーデータ）
- **設定**: `.env.local` でローカルSupabaseのURLを指定

### 本番環境
- **データベース**: 本番Supabase（`https://uheuaetcwqabmbpyhnio.supabase.co`）
- **データ**: 本番環境で実際に登録されたデータ
- **設定**: Vercelの環境変数で本番SupabaseのURLを指定

## 重要なポイント

**開発環境と本番環境は完全に分離されています。**

- ✅ **開発環境で登録したデータ** → 本番環境には反映されません
- ✅ **本番環境で登録したデータ** → 開発環境には反映されません
- ✅ **開発環境のデータ** → `supabase/seed.sql` のダミーデータのみ

## 選択肢

### オプション1: 開発環境でも本番Supabaseを使用する（推奨しない）

**メリット**:
- 本番環境のデータを開発環境でも確認できる
- データの同期が不要

**デメリット**:
- 開発中に本番データを誤って削除・変更するリスク
- 開発中のテストデータが本番環境に混入するリスク
- 開発環境の変更が本番環境に影響する

**設定方法**:
`.env.local` を以下のように変更：
```bash
NEXT_PUBLIC_SUPABASE_URL=https://uheuaetcwqabmbpyhnio.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=本番環境のANON_KEY
```

### オプション2: 本番環境のデータを開発環境にインポートする

**メリット**:
- 開発環境と本番環境を分離したまま、本番データを確認できる
- 開発中の誤操作が本番環境に影響しない

**デメリット**:
- データの同期が必要（定期的にインポートが必要）
- インポート手順が複雑

**手順**:
1. 本番Supabaseからデータをエクスポート
2. ローカルSupabaseにインポート
3. `supabase/seed.sql` を更新

### オプション3: 開発環境と本番環境を分離したままにする（現在の状態・推奨）

**メリット**:
- 開発中の誤操作が本番環境に影響しない
- テストデータで自由に開発・テストできる
- セキュリティが向上

**デメリット**:
- 本番環境のデータを開発環境で確認できない
- 開発環境で登録したデータは本番環境に反映されない

**現在の状態**:
- 開発環境: `supabase/seed.sql` のテストデータを使用
- 本番環境: 本番Supabaseの実際のデータを使用

## 推奨設定

**開発環境と本番環境を分離したままにすることを推奨します。**

理由：
1. **セキュリティ**: 開発中の誤操作が本番環境に影響しない
2. **データの整合性**: テストデータで自由に開発・テストできる
3. **ベストプラクティス**: 開発環境と本番環境の分離は一般的なベストプラクティス

## 本番環境のデータを確認したい場合

本番環境のデータを確認したい場合は、以下の方法があります：

1. **Supabaseダッシュボード**: 本番Supabaseのダッシュボードで直接確認
2. **本番環境のURL**: `https://www.skateright.io/` で本番環境を確認
3. **一時的に本番Supabaseを使用**: `.env.local` を一時的に変更（注意が必要）

## 開発環境で登録したデータを本番環境に反映する方法

開発環境で登録したデータを本番環境に反映するには：

1. **手動で本番環境に登録**: 本番環境のURL（`https://www.skateright.io/`）で直接登録
2. **データベースのエクスポート/インポート**: 開発環境からエクスポートして本番環境にインポート（上級者向け）

## まとめ

- **開発環境のスポット**: `supabase/seed.sql` のダミーデータ（テスト用）
- **本番環境のスポット**: 本番Supabaseに実際に登録されたデータ
- **データの同期**: 開発環境と本番環境は完全に分離されているため、自動的に同期されない

開発環境で登録したデータを本番環境に反映するには、本番環境のURLで直接登録する必要があります。
