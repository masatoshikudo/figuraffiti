# 同じ場所に複数の記録がある場合のUI設計

## 問題

同じ場所（座標が近い）に複数の記録が登録された場合、現在の実装では1つの記録しか表示されません。

## 設計方針

記録主役の設計では、**シンプルさ**が重要です。複数の記録がある場合も、迷わずに閲覧できるUIが必要です。

## 提案1: リスト表示（推奨）

### 概要
ポップアップ内に複数の記録をリスト表示し、クリックで詳細を表示する。

### UI構造
```
┌─────────────────────────────┐
│  📍 この場所の記録 (3件)     │
├─────────────────────────────┤
│  [記録1] Kickflip - 2024年  │ ← クリックで詳細表示
│  [記録2] Ollie - 2023年     │
│  [記録3] BS 180 - 2022年    │
└─────────────────────────────┘
```

### メリット
- シンプルで直感的
- すべての記録が一目で見える
- 実装が容易

### デメリット
- 記録数が多い場合、スクロールが必要

---

## 提案2: スライダー/カルーセル

### 概要
複数の記録をスライドで切り替えられる。

### UI構造
```
┌─────────────────────────────┐
│  ← [記録1/3] →             │
│  Kickflip - 2024年          │
│  [詳細表示]                 │
└─────────────────────────────┘
```

### メリット
- 1つの記録に集中できる
- モバイルで使いやすい

### デメリット
- 他の記録が見えない
- 実装が複雑

---

## 提案3: タブ表示

### 概要
複数の記録をタブで切り替えられる。

### UI構造
```
┌─────────────────────────────┐
│ [記録1] [記録2] [記録3]     │
├─────────────────────────────┤
│ Kickflip - 2024年           │
│ [詳細表示]                   │
└─────────────────────────────┘
```

### メリット
- 切り替えが明確
- 記録数が少ない場合に有効

### デメリット
- 記録数が多い場合、タブが多くなる
- モバイルでは使いにくい

---

## 推奨: 提案1（リスト表示）

### 実装方針

1. **マップクリック時**
   - 同じ場所の記録をすべて取得
   - 記録数が1件の場合: 現在の`SpotWindow`を表示
   - 記録数が2件以上の場合: リスト表示のポップアップを表示

2. **リスト表示ポップアップ**
   - 各記録をカード形式で表示
   - 技名、年、スケーター（あれば）を表示
   - クリックで詳細表示（`SpotWindow`を開く）

3. **詳細表示**
   - 選択した記録の詳細を`SpotWindow`で表示
   - 前後の記録に移動できるボタンを追加（オプション）

### コンポーネント構造

```
SpotBottomSheet
  ├─ MultipleRecordsList (記録数が2件以上の場合)
  │   └─ RecordCard (各記録)
  │       └─ onClick → SpotWindow を開く
  └─ SpotWindow (記録数が1件、または詳細表示時)
      └─ 現在の実装
```

### 距離の閾値

同じ場所とみなす距離の閾値:
- 50メートル以内（既存の`SPOT_MERGE_CONFIG.DISTANCE_THRESHOLD`を使用）

---

## 実装の優先順位

1. **Phase 1**: リスト表示の実装
   - 同じ場所の記録を取得する関数
   - `MultipleRecordsList`コンポーネント
   - `RecordCard`コンポーネント

2. **Phase 2**: 詳細表示の改善
   - 前後の記録に移動するボタン（オプション）

---

## ユーザー体験フロー

1. ユーザーが地図上のピンをクリック
2. 同じ場所の記録が複数ある場合:
   - リスト表示のポップアップが表示される
   - 各記録がカード形式で表示される
3. ユーザーが記録をクリック
4. 選択した記録の詳細が`SpotWindow`で表示される
5. 閉じるボタンでポップアップを閉じる

---

## 注意点

- 記録数が非常に多い場合（10件以上）の対応
  - 最初の5件を表示し、「もっと見る」ボタンで残りを表示
  - または、スクロール可能なリストにする

- パフォーマンス
  - 同じ場所の記録を取得する処理を最適化
  - 距離計算の負荷を考慮
