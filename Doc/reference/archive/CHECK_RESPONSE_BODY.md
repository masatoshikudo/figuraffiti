# レスポンスボディの確認方法

## 現在の状況

ブラウザの開発者ツールでサインアップリクエストの詳細を確認できています。リクエストは成功しています（ステータス200）。

しかし、**レスポンスボディ**を確認する必要があります。これがメール送信の成否を判断する重要な情報です。

## レスポンスボディの確認方法

### 1. ブラウザの開発者ツールで確認

1. ブラウザの開発者ツールを開く（F12）
2. **Network**タブを確認
3. `/signup`リクエストをクリック
4. **Response**タブをクリック
5. レスポンスボディを確認

### 2. 確認すべき情報

レスポンスボディに以下の情報が含まれているか確認してください：

#### 正常な場合（メール確認が必要）

```json
{
  "user": {
    "id": "ユーザーID",
    "email": "user@example.com",
    "email_confirmed_at": null,  // null = メール確認が必要
    "created_at": "2025-12-30T05:34:48.000Z"
  },
  "session": null  // null = メール確認が必要
}
```

#### エラーの場合

```json
{
  "user": null,
  "session": null,
  "error": {
    "message": "エラーメッセージ",
    "status": 400
  }
}
```

### 3. メール送信の成否を判断する方法

#### メール送信が成功した場合

- `user`オブジェクトが返される
- `session`が`null`になる（メール確認が必要な場合）
- `email_confirmed_at`が`null`になる（メール確認が必要な場合）
- エラーメッセージがない

#### メール送信が失敗した場合

- `user`オブジェクトが返されるが、エラーメッセージが含まれる可能性がある
- Supabaseのログにエラーメッセージが記録される
- Resendダッシュボードにログが表示されない

## よくある問題

### 問題1: レスポンスボディに`user`が含まれているが、メールが届かない

**原因**: 
- メール送信が試行されていない（SMTP設定が無効）
- メール送信が失敗している（SMTP接続エラー）

**確認方法**:
- Resendダッシュボードの **Logs** を確認
- Supabaseの **Authentication** → **Logs** を確認（可能な場合）

### 問題2: レスポンスボディにエラーメッセージが含まれている

**確認方法**:
- エラーメッセージの内容を確認
- エラーメッセージに基づいて対処

## 次のステップ

1. **ブラウザの開発者ツールでレスポンスボディを確認**（最重要）
2. レスポンスボディの内容を共有してください
3. Resendダッシュボードの **Logs** を確認
4. エラーメッセージがあれば、その内容を共有してください

## 重要

レスポンスボディには、メール送信の成否を判断する重要な情報が含まれています。

**ブラウザの開発者ツールのNetworkタブで、`/signup`リクエストをクリックして、Responseタブでレスポンスボディを確認してください。**
