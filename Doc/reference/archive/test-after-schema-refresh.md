# スキーマキャッシュリフレッシュ後のテスト手順

## 実行済み

✅ SQL Editorで `NOTIFY pgrst, 'reload schema';` を実行

## 次のステップ

### 1. 記録を投稿してテスト

1. ブラウザで `https://www.skateright.io/mapping` を開く
2. 地図上をクリックして仮ピンを立てる
3. ログイン（未ログインの場合）
4. トリック名とメディアURLを入力
5. 「記録を送信」ボタンをクリック

### 2. エラーが解消されたか確認

#### ✅ 成功の場合
- トースト通知に「投稿を受け付けました。承認後に公開されます。」と表示される
- ブラウザのコンソールにエラーが表示されない

#### ❌ まだエラーが出る場合
ブラウザのコンソールで以下のエラーが表示される可能性があります：

- `Could not find the 'address' column` → スキーマキャッシュがまだ更新されていない
- その他のエラー → 別の原因の可能性

### 3. まだエラーが出る場合の対処法

#### 方法A: もう一度スキーマキャッシュをリフレッシュ

```sql
NOTIFY pgrst, 'reload schema';
```

数秒待ってから再度試してください。

#### 方法B: Supabaseダッシュボードからリフレッシュ

1. Supabaseダッシュボード → **Settings** → **API**
2. **Reload schema** ボタンをクリック

#### 方法C: データベースのカラムを再確認

```sql
-- addressカラムが存在するか確認
SELECT column_name, data_type, is_nullable
FROM information_schema.columns
WHERE table_name = 'spots' AND column_name = 'address';
```

結果が返ってくれば、カラムは存在します。

#### 方法D: 一時的な回避策（コード側で対応済み）

現在のコードでは、`address`フィールドが`undefined`の場合は送信しないため、スキーマキャッシュエラーを回避できます。

ただし、`address`フィールドを保存したい場合は、スキーマキャッシュを正しくリフレッシュする必要があります。

## 確認事項

- [ ] 記録の投稿が成功したか
- [ ] ブラウザのコンソールにエラーが表示されていないか
- [ ] トースト通知が正しく表示されたか

## 次のアクション

テスト結果を共有してください：
- ✅ 成功した場合 → 問題解決
- ❌ まだエラーが出る場合 → エラーメッセージを共有してください
